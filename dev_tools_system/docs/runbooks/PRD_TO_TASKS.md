# PRD to Tasks

## Назначение
Этот runbook описывает, как превращать требования из PRD в backlog задач для разработки MultiKI.

## Главный принцип
Источник требований:
- docs/PRD.md

Источник исполнения:
- GitHub Issues
- GitHub Project

Codex не должен получать задачу напрямую из PRD.
Сначала требования из PRD должны быть разложены на issue и task packets.

## Когда использовать
Использовать:
- в начале спринта
- при появлении новой крупной фичи
- при изменении scope MVP
- при переразбиении уже существующего backlog

## Порядок работы

### 1. Выбрать секцию PRD
Нужно определить конкретную секцию PRD, которая будет переведена в задачи.

Примеры:
- MVP scope
- parser module
- planner module
- daily flow
- memory flow
- reflection flow

### 2. Дать задачу ChatGPT
Нужно попросить ChatGPT разложить секцию PRD на рабочие задачи.

Запрос должен включать:
- секцию PRD
- ограничения
- требование test-first
- просьбу выдать issue backlog

### 3. Получить на выходе
На выходе должны быть:
- список issue
- краткая цель каждой задачи
- затронутые файлы/зоны
- Definition of Done
- test contract
- branch name
- приоритет

### 4. Перенести в GitHub
Каждая задача переносится в:
- GitHub Issue
- GitHub Project

### 5. Выполнять по одной
В работу берётся только одна задача за раз.
Для каждой задачи потом создаётся отдельный task packet для Codex.

## Формат запроса к ChatGPT

Пример:

Возьми секцию PRD ниже и разложи её на issue для разработки.

Что нужно на выходе по каждой задаче:
- title
- goal
- context
- affected files/modules
- Definition of Done
- test contract
- suggested branch name
- priority

Ограничения:
- локальный стек
- минимальная сложность
- Codex должен работать test-first
- задачи должны быть достаточно маленькими для одного PR

## Правила качества
Хорошее разбиение на задачи:
- не смешивает несколько больших целей в одном issue
- позволяет сделать test-first реализацию
- имеет понятный DoD
- привязано к одной секции PRD
- может быть закрыто одним PR

Плохое разбиение:
- слишком большие задачи
- vague формулировки
- нет test contract
- нет DoD
- нет привязки к PRD section

## Результат
После выполнения этого runbook у тебя должен появиться рабочий backlog задач, готовых для переноса в GitHub Project и последующей реализации через Codex.
