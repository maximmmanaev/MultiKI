# MultiKI Dev Tools System

## Назначение
Эта папка содержит систему разработки MultiKI.
Она не является частью продуктового рантайма.
Она нужна для управления разработкой, документацией, задачами, ролями агентов и качеством изменений.

## Главные принципы
1. Источник истины по коду и документации — git-репозиторий.
2. Источник истины по задачам и статусам — GitHub Project.
3. Obsidian используется как визуальный слой поверх markdown-документов.
4. NotebookLM используется как исследовательский и аналитический слой по документам и внешним источникам.
5. Codex CLI работает как исполнитель изменений в коде и скриптах.
6. ChatGPT используется для декомпозиции, планирования, архитектуры и контроля процесса.
7. Ни одно важное решение не должно жить только в чате.

## Состав системы
- docs/ — основная документация системы разработки
- ops/obsidian-vault/ — vault для Obsidian
- ops/notebooklm/ — материалы для NotebookLM
- scripts/ — verify и служебные скрипты
- .github/ISSUE_TEMPLATE/ — шаблоны задач
- AGENTS.md — правила работы Codex и других агентов

## Рабочий цикл
1. Задача создаётся в GitHub Project / Issue.
2. Контекст задачи фиксируется в документации.
3. ChatGPT помогает разбить задачу и уточнить решение.
4. Codex меняет файлы в репозитории.
5. Codex запускает проверки.
6. Пользователь проверяет diff и только потом делает commit.
7. После merge документация обновляется.

## Ограничения
- Никаких автоматических commit/push без подтверждения пользователя.
- Любое изменение кода должно сопровождаться проверками.
- Любое архитектурное изменение должно отражаться в docs/ или adr/.

## Текущее состояние
Система разработки разворачивается поэтапно.
Сначала создаётся структура, затем правила работы, затем канбан, затем verify-контур и CI.



## Подключение локального pre-push hook

После клонирования репозитория нужно подключить локальный pre-push hook вручную:

    mkdir -p .git/hooks
    cp dev_tools_system/git-hooks/pre-push .git/hooks/pre-push
    chmod +x .git/hooks/pre-push

Этот hook запускает:

    ./dev_tools_system/scripts/verify.sh

Если verify падает, push должен быть остановлен.
