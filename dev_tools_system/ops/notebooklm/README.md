# NotebookLM в системе разработки MultiKI

## Роль NotebookLM
NotebookLM используется как исследовательский и аналитический инструмент.

Он нужен для:
- разбора проектной документации
- анализа внешних технических источников
- формулирования рисков, гипотез и вопросов
- подготовки кратких аналитических выжимок

## Что не является задачей NotebookLM
- не является источником истины по проекту
- не заменяет GitHub Project
- не заменяет git-репозиторий
- не заменяет Obsidian

## Рекомендуемые notebook'и
1. MultiKI Core Docs
2. Sprint Research
3. External Tech Docs

## Источники для загрузки
- markdown-файлы из docs/
- ADR
- sprint docs
- внешние URL
- PDF-документы
- официальные документации

## Правило работы
Любой полезный вывод из NotebookLM должен быть перенесён обратно в git-репозиторий:
- в docs/
- в adr/
- в sprint docs
- в summaries/

Нельзя хранить важные решения только внутри NotebookLM.
